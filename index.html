AI 에이전트와 잘 협업하는 법

✅ AI 에이전트가 잘 하는 것

  • 간단한 앱 — "해줘" 한 마디로 뚝딱
  • 코드 오류 스스로 찾아서 수정
  • 반복적인 작업 자동 처리
  • 디자인 수정·기능 추가 즉시 반영

💡 방향을 잘 주면 더 잘 된다

  • 사용자가 2종류면 — 각 역할을 명확히
  • 데이터가 공유되면 — 구조를 미리 정의
  • 화면이 여러 개면 — 흐름을 먼저 그리기
  • 원하는 결과가 구체적일수록 — 더 정확히 나옴
AI 에이전트는 이제 정말 잘 한다.
단, 어디로 가야 할지 모르면
열심히 엉뚱한 방향으로 간다.

바이브 코딩의 핵심은
AI를 막는 것이 아니라

바이브 코딩 — 두 가지 핵심 개념

① 먼저 설계, 그 다음 AI

Spec-Driven Development

코드를 짜기 전에 "뭘 만들지"를 먼저 문서로 정리하고,
그 문서를 AI에게 넘겨서 코드를 만들게 한다

건축으로 비유하면:

설계도(PRD) 없이 집 짓기 시작하면?
공사 중간에 전부 다시 해야 한다!

② AI에게 잘 전달하기

Context Engineering

"AI에게 얼마나 정확하게 설명하느냐"가
결과물의 품질을 결정한다

직원에게 업무 지시하기:

"알아서 해줘" vs "이 기획서 보고 1번부터 해줘"
결과가 완전히 다르다!

설계서 — 이렇게 작성하면 됩니다

📋 설계서 4가지 항목

1

한 줄 소개

2

사용자 종류와 역할

3

화면별 기능 목록

4

저장할 데이터

💡 완벽하지 않아도 됩니다.
AI가 읽고 질문할 수 있을 정도면 충분.

📄 음악 퀴즈 앱 설계서 예시

1. 서비스 소개 AI가 음악 퀴즈를 자동 생성하는 플랫폼. 교수자가 만들고, 학습자가 풀며 경쟁한다. 2. 사용자 - 교수자: 닉네임 + 비밀번호 입력, 퀴즈 생성·검토·공개 - 학습자: 닉네임 입력, 퀴즈 선택·풀기 3. 화면별 기능 [입장] 닉네임 입력 (교수자: 비밀번호 추가 확인) [교수자] 장르·난이도 설정 → AI 생성 → 검토 → 저장 [학습자] 퀴즈 목록 → 선택 → 풀기 → 점수 확인 [공통] 퀴즈별 실시간 리더보드 4. 저장 데이터 (Firebase Firestore) - quizzes: 제목, 장르, 공개여부, 문제 배열 - scores: 퀴즈ID, 닉네임, 점수, 제출시각 - sessions: 닉네임, 역할

좋은 프롬프트 공식 — Context Engineering

📐 프롬프트 4요소

① 나는 누구인가 — 비전공자, 처음 만들어봄

② 무엇을 만드나 — 설계서 전체 (역할·기능·데이터)

③ 지금 뭘 원하나 — 단계별 지시, 지금은 1단계만

④ 제약 조건 — 아직 코드 만들지 마 / 단계별로 만들어줘

💬 이렇게 시작하면

"음악 퀴즈 앱 만들어줘"

→ 뭔가 나오긴 함 · 역할 구분 없음 · 수정하다 꼬일 수 있음

✅ 4요소 갖추면

"나는 비전공자야. ← ① 음악 퀴즈 앱을 만들려고 해. [설계서 붙여넣기] ← ② 아직 코드 만들지 마. ← ④ 설계서 읽고 질문해줘." ← ③

💡 AI가 질문을 돌려준다 = 좋은 프롬프트의 신호

PART 1 · 바이브 코딩

설계서 있을 때 vs 없을 때 — 실제 차이

❌ 설계 없이 시작
나: "음악 퀴즈 앱 만들어줘" AI: (앱을 만들어줌) 나: "교수자 기능도 넣어줘" AI: (다시 만들어줌) 나: "리더보드도" AI: (또 다시...) 나: "아 처음이랑 구조가 달라졌어" AI: "처음부터 다시 할까요?"

요청할수록 구조가 꼬이고, 수정할수록 시간이 늘어남

✅ 설계서 먼저 작성
나: "설계서 읽고 질문해줘. 코드 아직 X" AI: "교수자 비밀번호를 어떻게 관리할까요?" 나: (답변) AI: "Firestore 컬렉션 구조 확인해도 될까요?" 나: (답변) AI: "1단계 만들겠습니다." → 완성

AI가 전체 구조를 파악한 뒤 만들어서
중간에 꼬이는 경우가 훨씬 줄어듦

설계서 작성 시간 30분 vs 수정 반복 시간 수 시간

완성 후 공개하기 — Share vs Publish

🔗

Share (링크 공유)

  • AI Studio 상단 Share 버튼 클릭
  • 링크 생성 → 카카오톡·이메일로 전송
  • 받은 사람은 구글 계정으로 접속
  • 서버 비용 없음

⚠️ 주의

화면 상단에
"실험용 앱" 문구 표시됨
→ 테스트·발표용으로 적합

✅ 추천 — 입문자용
🚀

Publish (정식 배포)

  • AI Studio → Deploy 버튼 클릭
  • Google Cloud Run에 자동 배포
  • 고정 URL 생성 → 누구나 접속 가능
  • Google 계정 없어도 사용 가능

⚠️ 주의

Google Cloud 결제 계정 필요
사용량에 따라 비용 발생 가능

심화 — 외부 공개용
🐙

GitHub + Vercel

  • AI Studio → Export to GitHub
  • Vercel에서 GitHub 연결 → 자동 배포
  • "실험용 앱" 문구 없음
  • 독립적인 URL로 운영

💡 장점

가장 깔끔하게 배포
포트폴리오·외부 공개 서비스로 최적

심화 Antigravity

지금 당장은 Share(링크 공유)로 충분 — 배포는 나중에 천천히
